Model Training Layer:

Overview:

The Model Training Layer is responsible for building machine learning models that predict the probability of customer churn.

This module consumes features from the feature store, trains multiple candidate models, evaluates their performance, and registers the best-performing model into the model registry for production inference.

The goal of this layer is to ensure that the deployed model is data-driven, reproducible, and properly versioned.

==============================================================
Objectives:

* The training layer performs the following responsibilities:
* Load training data from the feature store
* Prepare model input features
* Train multiple machine learning models
* Evaluate model performance using standard metrics
* Select the best-performing model
* Register the selected model in the model registry

This process ensures that only the most reliable model is used in the production pipeline.
=============================================================
Input Dataset:
The training pipeline loads the feature dataset generated by the feature engineering layer.

Input location:
"data/features/churn_features_v1.parquet"

The dataset contains:
* engineered features
* metadata columns
* the target variable (churn)

During training, metadata columns are removed so that only model-relevant features are used.
==============================================================
Training Data Preparation:
Before model training begins, the pipeline performs several preprocessing steps.

Feature Selection:
Columns not used by the model are removed, including:
* customerid
* snapshot_date
* pipeline metadata fields

The remaining columns become the model feature matrix X, while the churn column becomes the target variable y.
==============================================================
Train-Test Split:
The dataset is split into training and evaluation subsets using a stratified split.

Purpose:
* preserve class distribution
* ensure fair model evaluation

Typical split:
Training Data : 80%
Test Data     : 20%
===============================================================
Models Implemented:
The training pipeline evaluates multiple machine learning algorithms.

Logistic Regression:
Logistic Regression is used as a baseline model.

Advantages:
* simple and interpretable
* fast training
* provides a performance benchmark
-------------------------------------------------------
Random Forest:
Random Forest is an ensemble learning method that builds multiple decision trees and aggregates their predictions.

Advantages:
* robust to overfitting
* handles nonlinear relationships
* strong performance on tabular data.
----------------------------------------------------------
XGBoost:
XGBoost is a gradient boosting algorithm optimized for high performance.

Advantages:
* strong predictive power
* efficient training
* widely used in production ML systems
===========================================================
Model Evaluation:
Each trained model is evaluated using several metrics.

Evaluation Metrics
Metric	    Description
Accuracy	overall prediction correctness
Precision	proportion of correct positive predictions
Recall	    ability to detect actual churners
ROC-AUC	    ability to distinguish churn vs non-churn

In churn prediction systems, recall and ROC-AUC are especially important because the goal is to identify customers at risk of leaving.
=========================================================
Model Selection:
After evaluation, models are compared using their performance metrics.
The model with the best overall performance is selected as the production model.

For this project, the selected model is:
"Random Forest"
because it achieved the highest ROC-AUC and strong recall.
========================================================
Model Registry:
Once the best model is selected, it is registered in the model registry.

The registry stores:
* model version
* model file path
* training timestamp
* evaluation metrics
* feature version
* dataset metadata

The model artifact is stored in:
"artifacts/models/ "
